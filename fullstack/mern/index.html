    TaskSphere

    Microservices-based task management — Spring Boot, Spring Cloud, Docker

    Major Project
    Demo Source
    Problem: Monolithic task management apps lack independent deployability and fault isolation. Enterprise teams need user, task, and auth services operating independently. Existing solutions (Jira, Trello) are SaaS-only — not designed for teams that want self-hosted, modular task management with independent service scaling.
    Architecture
  • 3 independent microservices communicating via REST (Spring Cloud Feign) for loose coupling
  • Spring Cloud Gateway handles centralized routing, request filtering, and load distribution
  • Per-service database strategy enables independent scaling and fault isolation
  • Docker Compose for consistent multi-environment deployment
    • Engineering Decisions
  • Data consistency: Accepted eventual consistency across services rather than distributed transactions — used application-level validation and idempotent API design
  • Inter-service communication: Chose synchronous REST calls over async messaging for simplicity at current scale; architecture supports migration to event-driven patterns
  • API design: Versioned endpoints (/api/v1/) with pagination, filtering, and Swagger/OpenAPI documentation
    • Spring Boot 3 Spring Cloud Docker MySQL React JWT Feign Client
    Outcome: Production-deployed microservices system. Detailed schema, API docs, and deployment config on GitHub.
